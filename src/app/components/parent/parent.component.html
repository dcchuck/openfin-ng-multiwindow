OpenFin Angular Multi-window POC

Steps to Reproduce:

  1. Type anything on Component's A input box
  2. Notice how what you type is instantly displayed on Component B

  3. Now click on the "Popout Component B" button and move it to another monitor (or visible area)
  4. Go back to the main window and type anything on Component's A input box (repeating step 1)
  5. Notice how the poped out window (Component B) is not displaying what you are typing on Component A

Challenge: when you open a new window you basically bootstrap a brand new angular app.

It would be amazing if somehow OpenFin's platform is able to overcome this by making so you can popout child windows while keeping the session/context transparently (i.e. without having to update our code to use a communication system like OpenFin's InterApplicationBus or our own in-house built EventListener).

In case you guys are able to overcome this, please submit a PR so we can check it out.
